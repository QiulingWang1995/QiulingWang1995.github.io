实验三 使用Github管理Android项目

一、实验目的

1.掌握Android Studio中Git使用

2.掌握2.Github管理项目

二、实验任务

1.在Android Studio中新建项目

2.添加VCS中的Git支持

3.发布项目到Github

4.在Android Studio中进行修改代码、提交、新建分支、合并等操作

5.推送项目到Github

6.如有多人合作，处理pull request

三、实验步骤与结果

1.在Android Studio中新建项目

打开Android Studio,选择菜单项File->New Project，新建一个名为NerdLaunch的Android Studio项目，目标设备为Phone and Tablet，新建名为NerdLaunchActivity的空Activity。

2.添加VCS中的Git支持

点击菜单栏的VCS->VCS Operations Popup，选择Create Git Repository。

3.发布项目到Github

1）.安装好git后启动Androidstudio，打开如下路径File->Settings->Version Control(展开)->git在Path to Git executable后面的输入框输入你安装的git路径，点击test按钮如果出现 Git executed successfully 对话框说明配置成功，同时对话框会显示你安装的git版本号。



2).配置github登录信息：File->setting->Github，Login填账号如我的：QiulingWang1995，Password填登录密码。填完之后点击test按钮，如果出现如下对话框说明配置成功


3).接下来点击Android studio菜单中的“VCS”的选项，弹出的下拉的菜单中进行选择“import into version control”的选项，弹出的下一级的菜单中，进行选中“share project on github”的选项，如果在github仓库无上传过当前项目，需要进行创建一个github的昵称，new repository name的昵称，然后点击“share”，在commit message提交备注的信息，或是进行更新的项目的信息，点击“ok”。这样Android studio的左侧底部的位置中看到上传的成功的提示的信息。登录GitHub可查看到新的Repository NerdLaunch已经生成。



4.在Android Studio中进行修改代码、提交、新建分支、合并等操作

1）.修改代码并提交：修改代码后右击选择Git->Commit File进行提交。


2).新建分支：右下角点击master，选择New Branch，(或者点击右键，选择Git—>Repository—>Branches)新建本地分支dev_test，但远程仓库还没有dev_test分支。接下来把新建的dev_test本地分支推送到远程仓库：右击选择Git->Repository->Push,成功后远程仓库有了dev_test分支。


3).提交分支修改：对dev2_test分支修改之后，想要提交修改，在自己的分支上做如下操作：Git->Commit Directory>commit and Push->Push。

4).分支的切换：从当前dev2_test分支切换到dev_test分支。如右图点击右下角dev2_test分支选择dev_test分支，点击checkout则切换成功。

5).分支的拉取：Git->Repository->Pull->选择dev2_test分支->Pull。



6).分支的合并：Git->Repository->Branches，点击某一个分支。就可以选择切换到选择的分支，创建并切换到新的分支，和选择的分支做对比，和选择的分支合并，删除选择的分支操作。


5.推送项目到Github

接下来让我们创建个新文件

创建后 AndroidStudio会弹出下面对话框 这时候要点击Yes

再修改一下NerdLaunchActivity.java ，作为测试用。点击主面板上的 Commit Changes 按钮并填写提交信息，点击commit ，继续点击commit。然后Git->Repository->Push，将项目push到 Github。网页端显示已经上传到Github上了。

四、实验小结

通过本次使用Github管理Android项目的实验，掌握了Android Studio中Git使用和Github管理项目，又学习到新的方法管理Github项目。

3.时光穿梭

Git is a distributed version control system.
Git is free software.

再次提交$ git add readme.txt，在执行第二步git commit之前，我们再运行git status看看当前仓库的状态，再提交$ git commit -m “add readme”，提交后，我们再用git status命令看看仓库的当前状态，Git告诉我们当前没有需要提交的修改，而且，工作目录是干净（working directory clean）的。

4.远程仓库

1)身份认证
当本地Git仓库与Git远程仓库进行通信时需要通过SSH进行身份认证。
创建SSH Key
打开Shell（Windows下打开Git Bash），创建SSH Key：

你需要把邮件地址换成你自己的邮件地址，然后一路回车，使用默认值即可，由于这个Key也不是用于军事目的，所以也无需设置密码。
如果一切顺利的话，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥用于验证自己的身份，id_rsa.pub是公钥用在Git远程服务器上表明自己的身份。
登陆GitHub，打开“Account settings”，“SSH Keys”页面：
然后，点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容：点“Add Key”，你就应该看到已经添加的Key。

在Github上用Markdown创建技术博客

实验一 在Github上用Markdown创建技术博客

一、实验目的

1.熟悉Github网站

2.掌握Markdown语法

二、实验任务

1.注册Github账号

我的账号：QiulingWang1995

2.创建Github Pages repository

3.发布第一篇博客


4.学习Markdown

5.利用hexo生成博客并部署到github

三、实验步骤与结果

一、本地配置hexo

1 安装node-v7.7.1-x86.msi

2. 安装Git-2.12.0-32-bit

3. 设置npm镜像为淘宝，http://npm.taobao.org/

1 cnpm install -g cnpm –registry=https://registry.npm.taobao.org

4. 安装Hexo

1 cnpm install –g hexo -cli

5. 新建文件夹MD到D盘根目录，复制地址D：/MD

1 cd /d D：/MD

6.初始化站点

1 hexo init hexo
2 cd hexo
3 cnpm install

7.浏览本地站点

1 hexo server

二、在GitHub上部署Hexo

1.创建一个Github Pages repository

2.修改配置文件——_config.yml（Hexo文件根目录下）

1 type: git
2 reposutoty: https://github.com/QiulingWang1995/QiulingWang1995.github.io.git
3 branch: master

3.配置site部分（可选）

1 # Site
2 title: 汪秋凌的博客
3 subtitle: 自建博客步骤
4 description:
5 author: 汪秋凌
6 language:
7 timezone:

4. 如要要修改主题，将主题文件克隆（clone）到themes目录下，并修改_config.yml配置文件中themes参数（如landscape改为jacman）。

5. 安装 hexo-deployer-git

1 npm install hexo-deployer-git –save

6.配置git全局邮箱与名字

1 git config –-global user.email “ql.wang@gmail.com”
2 git config –-global user.name “Qiuling Wang”

7.生成博客

1 hexo generate

8.部署博客

1 hexo server

三、添加新文章

1.生成新文章

1 hexo new newblog

2.用编辑器修改文章

2 myhexo\source_posts\newblog.md

3.生成并发布到Github

3 hexo d –g

四、实验小结

通过学习搭建博客平台，对博客平台和Markdown开发文档有了基础的了解。
