实验三 使用Github管理Android项目

一、实验目的

1.掌握Android Studio中Git使用

2.掌握2.Github管理项目

二、实验任务

1.在Android Studio中新建项目

2.添加VCS中的Git支持

3.发布项目到Github

4.在Android Studio中进行修改代码、提交、新建分支、合并等操作

5.推送项目到Github

6.如有多人合作，处理pull request

三、实验步骤与结果

1.在Android Studio中新建项目

打开Android Studio,选择菜单项File->New Project，新建一个名为NerdLaunch的Android Studio项目，目标设备为Phone and Tablet，新建名为NerdLaunchActivity的空Activity。

2.添加VCS中的Git支持

点击菜单栏的VCS->VCS Operations Popup，选择Create Git Repository。

3.发布项目到Github

1）.安装好git后启动Androidstudio，打开如下路径File->Settings->Version Control(展开)->git在Path to Git executable后面的输入框输入你安装的git路径，点击test按钮如果出现 Git executed successfully 对话框说明配置成功，同时对话框会显示你安装的git版本号。



2).配置github登录信息：File->setting->Github，Login填账号如我的：QiulingWang1995，Password填登录密码。填完之后点击test按钮，如果出现如下对话框说明配置成功


3).接下来点击Android studio菜单中的“VCS”的选项，弹出的下拉的菜单中进行选择“import into version control”的选项，弹出的下一级的菜单中，进行选中“share project on github”的选项，如果在github仓库无上传过当前项目，需要进行创建一个github的昵称，new repository name的昵称，然后点击“share”，在commit message提交备注的信息，或是进行更新的项目的信息，点击“ok”。这样Android studio的左侧底部的位置中看到上传的成功的提示的信息。登录GitHub可查看到新的Repository NerdLaunch已经生成。



4.在Android Studio中进行修改代码、提交、新建分支、合并等操作

1）.修改代码并提交：修改代码后右击选择Git->Commit File进行提交。


2).新建分支：右下角点击master，选择New Branch，(或者点击右键，选择Git—>Repository—>Branches)新建本地分支dev_test，但远程仓库还没有dev_test分支。接下来把新建的dev_test本地分支推送到远程仓库：右击选择Git->Repository->Push,成功后远程仓库有了dev_test分支。


3).提交分支修改：对dev2_test分支修改之后，想要提交修改，在自己的分支上做如下操作：Git->Commit Directory>commit and Push->Push。

4).分支的切换：从当前dev2_test分支切换到dev_test分支。如右图点击右下角dev2_test分支选择dev_test分支，点击checkout则切换成功。

5).分支的拉取：Git->Repository->Pull->选择dev2_test分支->Pull。



6).分支的合并：Git->Repository->Branches，点击某一个分支。就可以选择切换到选择的分支，创建并切换到新的分支，和选择的分支做对比，和选择的分支合并，删除选择的分支操作。


5.推送项目到Github

接下来让我们创建个新文件

创建后 AndroidStudio会弹出下面对话框 这时候要点击Yes

再修改一下NerdLaunchActivity.java ，作为测试用。点击主面板上的 Commit Changes 按钮并填写提交信息，点击commit ，继续点击commit。然后Git->Repository->Push，将项目push到 Github。网页端显示已经上传到Github上了。

四、实验小结

通过本次使用Github管理Android项目的实验，掌握了Android Studio中Git使用和Github管理项目，又学习到新的方法管理Github项目。
